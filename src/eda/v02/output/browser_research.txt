S1: M-Score Threshold for Indian Equities

  This makes our gate overly conservative (higher false negative rate)

- At -2.22: fewer false positives but higher false negative rate
- India-specific: Shah et al. (2018) BSE 100 study — 11 of 65 flagged as

  because Indian companies often don't separately report COGS vs SG&A
- DHFL and Yes Bank are financial institutions — M-Score explicitly EXCLUDES them
- India Forensic estimates at least 1,200 listed companies have counterfeited
  financial statements

RECOMMENDATION:
- Fix threshold mismatch: use -1.78 with 8-var model, OR -2.22 with 5-var model
- Add fallback to 5-variable model when SGAI data is unavailable
- Keep current banking/NBFC exemption (M-Score not designed for financials)

S8: Worst-5-Day Stress Test Methodology
KEY FINDINGS:
- Institutional standard: "Time to Liquidate" (TTL) framework
- Normal conditions: 10-20% of ADV participation rate
- Stressed conditions: 5% of ADV participation rate
- IMF uses 1st percentile of net flows for worst-case calibration
- Zero-volume day analysis: LMx (standardized zero daily volumes) is one
  of the best liquidity proxies for emerging markets

RECOMMENDED METHODOLOGY:
1. Calculate 5th percentile daily volume over 6 months (125 trading days)
2. Days_to_exit = Position_size / (P5_daily_volume x 10% participation)
3. PASS: Days_to_exit <= 5 trading days
4. FAIL: Days_to_exit > 5 trading days
5. Zero-volume check: <= 2 zero-volume days in 6 months

RECOMMENDATION:
- Our current "worst 5 days in last 20" approach is a simplified version
- More rigorous: use P5 volume over 6 months (not worst 5 of last 20)
- Add zero-volume day count as supplementary check
- Current 0.3x multiplier is reasonable but P5 approach is more principled

Sources: BIS, IMF WP/17/226, Katalysys, AMF France, CSSF

S11: Missing Data Handling — Pass vs Fail
KEY FINDINGS:
- "MISSING DATA = PASS" IS NOT SAFE
- Bryzgalova et al. (2022): missing data is NOT random — it clusters
  cross-sectionally and over time
- Stocks with missing values have LOWER RETURNS than counterparts
- Survivorship bias: momentum strategy 26% CAGR (biased) vs 12.2% (unbiased)
- Professional quant systems (FactSet): 1,500+ automated data quality checks
- QuantConnect: if fundamental data unavailable, stock is EXCLUDED entirely

THREE APPROACHES:
| Approach    | Description                      | Risk                      |
|-------------|----------------------------------|---------------------------|
| Exclude     | Drop stocks with missing data    | Selection bias             |
| Pass        | Let missing through              | Distressed companies pass  |
| Penalize    | Assign neutral/worst quintile    | Small model risk           |

RECOMMENDATION:
- Current v0.2 "return True" for missing data is TOO LENIENT
- Switch to "conditional pass with penalty flag" — pass but tag as
  "data incomplete, lower confidence"
- For FORENSIC gates (M-Score, CCR, Pledge): strict FAIL on missing data
  is more defensible (those gates specifically screen for red flags)
- For EARNINGS gates: conditional pass + confidence reduction is appropriate
- Differentiate forensic gates (strict) from earnings gates (lenient)

Sources: QuantPedia, QuantRocket, FactSet, QuantConnect, ScienceDirect

S13: VAM Normalization
KEY FINDINGS:
- Standard pipeline: compute raw momentum → divide by volatility →
  cross-sectional z-score → winsorize at +/-3 sigma
- BSE Momentum Index: winsorize at +/-3 sigma, then
  score = 1 + z if positive, 1/(1-z) if negative
- LOW-VOLATILITY PROBLEM: dividing by tiny sigma creates extreme scores
- Solution: set minimum volatility FLOOR (e.g., 5th percentile of
  cross-sectional volatility)
- Van Zundert (SSRN): incorporates ex-ante volatility estimates
- Cross-sectional normalization is most common approach

RECOMMENDATION:
- Add skip-month (exclude most recent month — same as MRS)
- Divide raw 12-1 month return by 252-day realized volatility
- Cross-sectional z-score normalization
- Winsorize at +/-3.0
- Set minimum volatility floor at 5th percentile of cross-sectional vol
- This prevents utilities from getting artificially high VAM scores

Sources: Newfound Research, Van Zundert (SSRN), BSE Factor Indices

S17: MRS-VAM Factor Correlation (0.64 Spearman)
KEY FINDINGS:
- 0.64 correlation = 41% shared variance (0.64^2)
- Three approaches:
  1. ORTHOGONALIZE: regress VAM on MRS, use residual (removes shared variance)
     - Pitfall: order-dependent (arbitrary which factor "keeps" shared info)
  2. REDUCE COMBINED WEIGHT: cap total momentum allocation
  3. PCA COMPRESSION: extract PC1 as single "composite momentum" factor
- Two Sigma's Venn: does NOT residualize equity style factors like Momentum
- Robeco: uses residualization to correct stock returns for style factors
- NBER: stock momentum may just aggregate autocorrelations in other factors
- Quality-momentum correlation is low (~0.29) — good diversification

RECOMMENDATION:
- SIMPLEST APPROACH: compute single composite momentum score
  (e.g., average of normalized MRS and normalized VAM) with single weight
- ALTERNATIVE: residualize VAM against MRS (isolate volatility-adjustment
  component), keep both but with reduced total weight
- If keeping both: reduce combined weight from 55% to 40-45%
- Replace freed-up weight with less-correlated factor (value, quality gate)

Sources: ResearchGate, Two Sigma Venn, AQR, NBER, Robeco, Research Affiliates
